F3-M2.8 — Fix renderizado \\ en parrafos
latex-parser.ts + LatexBlock.tsx

Los bloques de parrafo con saltos de linea \\ (doble backslash LaTeX) ahora se renderizan como <br/> en el visor. Cambios aplicados en el parser y en el componente de renderizado de bloques.

// Antes: \\ era ignorado o causaba artefactos // Ahora: texto.split('\\\\') → joined con <br/> latex-parser.ts — procesado a nivel de parser LatexBlock.tsx — renderizado del fragmento con br
✓
Fix catch-all regex en LatexBlock.tsx
LatexBlock.tsx

El regex catch-all que capturaba comandos LaTeX desconocidos eliminaba el texto visible del argumento. Por ejemplo \textbf{algo} desaparecia en lugar de mostrar "algo". Corregido: ahora el texto del argumento se muestra como fallback cuando el comando no es reconocido.

✓
Fix hydration error en home/page.tsx
app/(app)/home/page.tsx

El uso de localStorage directamente en el inicializador de useState causaba un mismatch entre el HTML generado en servidor y el renderizado en cliente (hydration error). Migrado al patron seguro: valor inicial neutro en useState + lectura de localStorage en useEffect post-montaje.

// Antes (causa hydration error): const [val, setVal] = useState(localStorage.getItem('key')) // Ahora (patron SSR-safe): const [val, setVal] = useState(null) useEffect(() => { setVal(localStorage.getItem('key')) }, [])
