Visor PDF-like con perfiles de plantilla
lib/template-profiles.ts + LatexViewer refactor
  • Nueva interfaz TemplateProfile con geometria (margenes, columnas), tipografia (fuentes, tamanos), colores (accent, fondo, headings) y layout por plantilla.
  • Perfiles reales derivados de los preambles LaTeX de las 4 plantillas activas: lecture_notes, 2cols_portrait, landscape_3col_maths, study_form + perfil default.
  • LatexViewer muestra una hoja blanca A4 sobre fondo neutro oscuro, igual que un visor PDF real. El texto es negro sobre blanco (fix color: '#111111').
  • Zoom via transform: scale() con patron two-div: el div exterior fija el espacio visual (altura × escala), el interior aplica la transformacion. Resuelve el problema de scroll height con zoom negativo.
  • CSS custom properties (--accent, --heading-color, etc.) inyectadas por plantilla activa. Los section headings y theorem environments las consumen automaticamente.
🔧
Mejoras A–E al visor interactivo
5 mejoras independientes de calidad y UX
  • A — KaTeX macros compartidas: constante KATEX_MACROS extraida a lib/katex-macros.ts y reutilizada en LatexBlock y ChatPanel sin duplicacion.
  • B — Preview KaTeX en chips BlockReference: el chip que aparece en el chat cuando el usuario referencia un bloque ahora muestra el latex renderizado, no el codigo fuente crudo.
  • C — Contadores undo/redo en toolbar: la barra superior muestra cuantos pasos atras y adelante estan disponibles en tiempo real. Implementado con estado React (undoCount, redoCount) en lugar de refs.
  • D — Scroll automatico tras Apply: al aplicar una sugerencia de IA, el visor hace scroll automatico hasta el bloque modificado para que el usuario vea el resultado inmediatamente.
  • E — Fix replace Nth-occurrence: cuando dos bloques tienen el mismo latex_source identico, el replace anterior era fragil (reemplazaba la primera ocurrencia siempre). Nueva estrategia: busca la Nth ocurrencia del string por posicion de bloque en el documento.

Fixes del reviewer
4 bugs corregidos tras revision de codigo
  • Zoom wrapper height: la altura visual del wrapper exterior se calcula explicitamente como naturalHeight × scale en lugar de depender de marginBottom negativo (que era fragil y causaba solapamientos).
  • FormatToolbar dentro de !hideToolbar: la toolbar de formato se movia fuera del bloque condicional, apareciendo aunque el bloque no estuviera en modo edicion.
  • IDs duplicados en multicols: parseLatexInternal no resetea el contador _idCounter al re-parsear — los IDs de bloque eran diferentes entre renders, causando problemas de reconciliacion React y en el Nth-occurrence replace.
  • Boton redo usa redoCount state: el boton redo consultaba ref.current directamente en lugar del estado React, por lo que el UI no se actualizaba al cambiar.

AI edita el documento completo via prompts del chat
Flujo C: document-level AI edit con preview y confirm/discard

El usuario puede escribir en el chat cosas como "cambia el color de las ecuaciones a rojo" y la IA genera un nuevo latex_source completo del documento. El visor muestra un banner "AI Preview" con outline indigo y el usuario puede Aplicar o Descartar.

  • AIProvider — editDocument(): metodo nuevo con JSON mode. En un solo LLM call clasifica si el mensaje del usuario es una edicion del documento (devuelve nuevo latex_source) o una respuesta conversacional (devuelve texto). El clasificador esta en el prompt de sistema.
  • app-api — POST /latex/edit-document: nueva ruta que recibe el latex_source actual + el mensaje del usuario y devuelve el resultado clasificado.
  • app-web — POST /api/documents/[id]/chat-edit: route handler que guarda los mensajes en DB (historial de chat) y retransmite la respuesta de app-api.
  • LatexViewer — prop pendingDocumentEdit: cuando el estado es activo, el visor muestra un banner amarillo/indigo "AI Preview" y un outline en el documento para indicar que hay cambios pendientes de confirmar.
  • ChatPanel — DocumentEditPreviewCard: nuevo componente de preview con botones Apply y Discard. Al aplicar, llama al mismo flujo de persistencia que las ediciones de bloque (compilacion PDF + nueva version en DB).
  • page.tsx: estado pendingDocumentEdit y wiring completo entre ChatPanel y LatexViewer.
Decisiones tecnicas tomadas
Decision Que se eligio Por que
Clasificador AI (edicion vs chat) JSON mode en un solo LLM call dentro de editDocument() Evita dos round-trips (uno para clasificar, otro para editar). El modelo es suficientemente bueno en clasificar con el contexto del prompt de sistema.
Zoom en visor Two-div wrapper con altura calculada explicitamente El patron transform: scale() no altera el layout flow — el div exterior reserva el espacio visual correcto para que el scroll funcione.
IDs de bloque en multicols parseLatexInternal mantiene el contador entre llamadas Los IDs deben ser estables entre renders para que React reconcilie correctamente y el Nth-occurrence replace encuentre el bloque correcto.
KaTeX macros Constante compartida en lib/katex-macros.ts Eliminacion de duplicacion entre LatexBlock y ChatPanel. Una sola fuente de verdad para las macros (\dd, \real, \cplex).
Problemas encontrados
⚠
Calidad de AI document edits — parcial. La feature funciona: el usuario puede pedir cambios al documento y verlos en preview antes de confirmar. Sin embargo, la IA no siempre aplica el cambio a todas las instancias. Ejemplo: pedir "cambia el color de las ecuaciones a rojo" modifico solo algunas ecuaciones, no todas. Esto es un problema de calidad del prompt y estrategia de edicion, no un bug de arquitectura. Pendiente mejorar en proxima sesion o como parte de F4.
⚠
app-api Docker desactualizado. La imagen Docker de app-api no incluye el endpoint POST /latex/edit-document hasta que se rebuilde. Para usar la feature en local: npm run dev directamente en app-api/.
✓
Todos los demas fixes sin incidencias. Los 4 bugs del reviewer (zoom height, FormatToolbar, duplicate IDs, redo state) quedaron resueltos sin regresiones conocidas.
