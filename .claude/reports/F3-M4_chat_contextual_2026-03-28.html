F3-M4 — Chat Contextual

Edicion de bloques con IA directamente desde el visor interactivo

Sesion: 2026-03-28 · BetterNotes v2

Criterio de aceptacion cumplido: El usuario puede referenciar un bloque del visor con click derecho → ver el chip en el chat → escribir un prompt → recibir preview KaTeX del bloque modificado → pulsar "Apply" → el bloque se actualiza en el visor → al descargar el PDF, refleja los cambios.

Sub-milestones implementados

  • ✓ M4.1 Panel lateral de chat vinculado al visor. Sin referencia: guia al usuario a hacer click derecho. Con referencia: chip + input contextual.
  • ✓ M4.2 Chip visual con "x" encima del input. BlockReference: { blockId, blockType, latex_source, adjacentBlocks[] }.
  • ✓ M4.3 Endpoint POST /api/documents/[id]/edit-block (app-web) + ruta POST /latex/edit-block (app-api) + editBlock() en AIProvider/OpenAIProvider.
  • ✓ M4.4 Preview del bloque modificado con KaTeX renderizado en BlockEditPreviewCard. Soporte formula-block (display mode) y paragrafos con inline math.
  • ✓ M4.5 Botones "Apply" / "Discard". Apply: actualizacion optimista en LatexViewer via prop applyBlockEdit + token.
  • ✓ M4.6 Al Apply: reemplazar string exacto del bloque en el .tex completo, compilar con /compile-only, persistir nueva version en DB (document_versions).
  • ✓ M4.7 Undo/redo en memoria en LatexViewer: historial hasta 20 estados. Ctrl+Z (undo), Ctrl+Y / Ctrl+Shift+Z (redo). Pushea en confirm, applyFormat y applyBlockEdit.

Flujo de usuario

1 Click derecho sobre un bloque en el LatexViewer → menu contextual "Reference in chat"
↓
2 ChatPanel recibe BlockReference. Chip visual aparece sobre el input con type badge + preview LaTeX + boton X
↓
3 Usuario escribe prompt ("make this formula more explicit") y pulsa Enter
↓
4 POST /api/documents/[id]/edit-block → app-api /latex/edit-block → editBlock() en OpenAIProvider → devuelve modifiedLatex
↓
5 BlockEditPreviewCard muestra el bloque modificado renderizado con KaTeX + collapse "View LaTeX source"
↓
6 Usuario pulsa "Apply": actualizacion optimista en LatexViewer (bloque reemplazado visualmente)
↓
7 onLatexChange dispara → pendingApplyLatex → POST /edit-block (apply:true) → compile + persist nueva version → onApplyPersisted → reloadDocument

Decisiones tecnicas

DecisionValor
Granularidad de edicion Bloque completo — la IA recibe y devuelve el bloque entero, no fragmentos
Busqueda y reemplazo en .tex String.replace() exacto del latex_source del bloque — sin parser adicional
Actualizacion optimista LatexViewer actualiza el bloque visualmente antes de confirmar persist en DB
Historial undo/redo En memoria (ref array de Block[]), hasta 20 estados. Se resetea al cargar nueva fuente
Contexto al AI blockLatex + blockType + adjacentBlocks (±2) + fullLatex truncado a 3000 chars
Endpoint unificado POST /edit-block maneja tanto preview (sin apply:true) como persist (apply:true)
KaTeX en ChatPanel katex importado en ChatPanel.tsx con mismos macros custom (\dd, \real, \cplex)
